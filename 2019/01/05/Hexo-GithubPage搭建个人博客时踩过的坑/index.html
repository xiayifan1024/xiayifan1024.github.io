Hexo+GithubPage搭建个人博客时踩过的坑

0x000 前言 搭建博客初衷

很久之前想正经的写点什么，还记得在QQ空间写过文章，但总是不尽如人意。后来网上冲浪博客盛行，我也申请个新浪账号写过一段时间的博客，但是样式、广告之类的总感觉我的博客不是我的（控制欲hhh），自己建站又没钱没本事。就这样到了大学干过一年多的微信推送新媒体之后，我又开始打起了写博客的主意。正好有一天看到了Hexo+GithubPage建立博客的文章，让我有了一些想法。

写博客的意义是什么？我一直认为只有两件事：记录、分享。而微博、微信等社交平台的出现虽然也是能起到记录与分享的作用，但是对于一些深入的思考分析总是不够尽如人意。代码也好数学公式也好个人思考也好，放在那几英寸的屏幕上总是感觉累，于是就还是考虑用博客吧。

毕竟我也不是大牛，以现在的水平写出来的东西也过于简单与重复，很难起到分享知识的作用，与其说是博客，倒不如说是一些笔记之类的，记录一下自己的学习历程之类的，写一些方法与感想。乐呵乐呵。而且也算是通过写博客加强一下对知识的记忆吧。

线上：

Hexo官方管理器：
在git中安装hexo-admin
$ npm i hexo-admin --save
在之后键入命令
hexo s http://localhost:4001/admin/
可直接利用hexo内置的makedown进行编辑管理

简书、csdn、cmd Markdown等也都可以进行线上编辑

0x002 出现的问题及注意

因为是建站好久才写，所以其中辛苦忘了好多，想起多少写多少吧

0、注意Git与Nodejs的版本，Hexo对两者有一定的版本支持
1、在博客根目录下执行Git Bash
2、博客网址中必须使用你github的用户名，即某用户的用户名为ClearLove7，那么网址必须为ClearLove7.github.io
3、Hexo对空格特别敏感，在配置中请务必注意空格问题，如在tag: Hexo中冒号后存在空格。在配置文件及文章中忽视此问题可能会导致页面错误、文章无法上传、格式错误等错误。
4、Next6可能会存在 PageBuildFailure 的问题。考虑主题配置文件中的 ‘override’ 改成 ‘ture’。
5、注意文件路径，如头像图片为在’F:\blog\source\assets\img\xxx.jpg’其中 ‘blog’ 为根目录，则在配置头像图片时注意为’/assets/img/xxx.jpg’
